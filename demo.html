What is Vuex > "Centralized State Management for Vue.js"
## State?
> "In information technology and computer science, a program is described as stateful if it is designed to remember preceding events or user interactions; the remembered information is called the state of the system."
## State! > Basically => The stuff thats going on at a specific time (like now).
### Analogy - State of the Union: - Unemployment is 4.3% - Nasa found water on the moon - And the country is about to burn itself down
Why Manage State? (A lot of reasons... but in regards Vuex) > Generally, You're application needs to be aware of the rest of the stuff going on... specifically component to component communications
### Solutions for State Management - Prop / Event Emmiters - Event Bus - Vuex
### Prop / Event Emiters The process for passing state in between components and the App > (Great for a small application, bad for a BIG application)

Demo

Event Bus An Event Bus is a Separate Vue instance that you pass events to and from > (Really good for small-medium sized applications, bad for a BIG application, unnecessary for a small)

Demo

Vuex (i.e. the reason I'm here) > (good for everything that is neither small nor small-medium)
How to install > `npm or yarn install vuex`
How to implement - Add import statements to main.js import { store } from './store/store'; - Add store to Vue instance new Vue({ el: '#app', router, store, template: '', components: { App }, });
- Create store.js (do store stuff) import Vue from 'vue'; import Vuex from 'vuex'; Vue.use(Vuex); // eslint-disable-next-line export const store = new Vuex.Store({ state: { counter: 0, }, });
Access the Store computed: { storeCounter() { return this.$store.state.counter; }, },
